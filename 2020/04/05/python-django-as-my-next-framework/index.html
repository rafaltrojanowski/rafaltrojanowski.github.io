Python/Django as a interesting alternative for Ruby on Rails

Python Django - My first impressions

Recently I used Django for a weekend project. Django is getting more and more popularity for Web Development these days.
Thanks to the fact that Python is easy to learn and multi purpose language (Web Scrapping, Web Development and Artificial Inteligence)
and was used by Google for scrapping a sites and then indexing them it’s definietly worth to have look at it, if you haven’t tried it yet.

Here are my findings.

Installation

Installation is process is smooth. After instalations project structure looks minimalistic, which is cool.
The running server is really fast and there is no magic (if you have ever used spring, you may know what I mean).

Admin Dashboard

The choice what to use for Dashboard for db managment is quite obvious,
django-admin, so far pretty nice experience. I’m not going to do many customizations by the way.

Another thing that I love is how easy is data seeding. It literralyu takes seconds. You just need to run one command

Django is very inteligent framework. I saw heading “Framework for perfectionsist with a deadline”, which
sounds like nice slogan.

So what’s smart is the fact that when you generate admin account by this command:

The problems occurs when you have null false column and already existing data, but command line will
ask you what to do in that case, set default or … It looks like very developer friendly experience.

Forms

Form object generation is nice and docs describe the process well.
The only issues that I spoted were related to adding custom css classes and placeholders for inputs
but I easliy found and addon that handles this case.

Image upload.

It’s easy thanks to build in Image field. No additiuonal solutions needed for that common feature.
The problematice part is that you need to incliude it to static files to don’t get 404
